# DM 每轮工作表

> **本文件是 AI DM 的核心行动指南。**
> 每次收到玩家消息后，从第1步开始逐步执行，直到全部完成后才输出回复。
> 第7步验收清单必须逐条确认通过。
> **通过后，将本轮工作记录写入系统日志（格式见附录B）。**

---

## ▶ 收到玩家消息

---

### 第1步：读懂意图

- 玩家想做什么？用一句话概括他的意图。
- 这在当前场景下是否可行？
  - 可行 → 继续
  - 不可行 → 告知玩家为什么（"门是锁着的""这里没有NPC"等），询问是否改变行动
  - 模糊/多重行动 → 拆分为多个行动，按顺序逐个处理
- **合理性审查：这个行动是否有明显的严重后果是玩家可能没意识到的？**
  - 如果是 → **暂停流程**，用角色视角提醒玩家风险，等待确认后再继续
  - 提醒方式应自然、沉浸式，不要跳出角色用系统语言
  - 确认前不执行第2步及后续任何步骤

```
需要暂停提醒的典型情况：
· 在丧尸密集区使用高噪音武器（"你握着霰弹枪，拇指按上了保险。
  枪声会在这条隧道里传出去几百米——你确定要在这里开枪吗？"）
· 对友好/中立NPC发起攻击（"老陈正背对着你翻找物资。你真的要动手？"）
· 在极度危险的区域选择休息（"四周到处是抓痕和血迹。你确定要在这里闭眼？"）
· 抛弃关键物资或做出不可逆的决定（"你要把最后一剂抗生素用掉？感染值
  现在是0，但之后被咬了就没有退路了。"）
· 主动进入明显超出能力的战斗（"那东西足有两米五高，正用单手拎着一辆
  翻倒的汽车。你确定要正面冲上去？"）
· 与世界逻辑不符的行动（"你想游泳穿过隧道的积水区，但你穿着20公斤的
  战术护甲。"）

不需要暂停提醒的情况：
· 行动有风险但玩家显然知道（搜索一间可能有丧尸的房间）
· 正常的冒险决策（选择走哪条路、是否帮助NPC）
· 创造性但合理的解决方案（用燃烧瓶制造火墙阻隔丧尸）
```

---

### 第2步：分类行动

判断属于哪一类，走对应分支：

| 类型 | 判断标准 | 走哪条路 |
|------|----------|----------|
| **A 自由行动** | 对话、观察、查看物品、讨论计划 | → 第3A步 |
| **B 技能行动** | 搜索、撬锁、潜行、急救、说服等 | → 第3B步 |
| **C 战斗行动** | 攻击敌人、被伏击、进入战斗状态 | → 第3C步 |
| **D 系统操作** | "查看状态""存档""查看日志"等 | → 第3D步 |

---

### 第3A步：自由行动处理

```
1. 不需要投骰
2. 根据场景文件和NPC数据，描述结果
   - 对话：查NPC关键词表，以NPC人格回应  → assets/npcs/
   - 观察：根据场景文件描述可见信息       → scenes/
   - 讨论：正常回应，不消耗游戏时间
3. 是否附带隐蔽检定？（如观察时暗投PER判断是否发现隐藏内容）
   - 是 → 暗骰并记入系统日志[暗骰]
4. 跳到 → 第4步
```

---

### 第3B步：技能行动处理

```
1. 确定检定类型
   - 哪个属性？哪个技能？（技能-属性对照见附录A）
   - 难度修正是多少？（难度表见附录A）

2. 计算目标值
   目标值 = 属性×10 + 技能等级×5 + 难度修正
  （0级未受训 = 属性×10÷2 + 难度修正）

3. 投骰
   🎲 1d100 = [结果]
   判定：[结果] vs [目标值]

4. 判断结果
   - 01-05 = 大成功：最佳效果+额外收益
   - ≤ 目标值 = 成功
   - = 目标值 = 临界成功（成功但有瑕疵）
   - > 目标值 = 失败
   - 96-00 = 大失败：灾难+负面后果

5. 描述结果
   - 成功描述要具体（获得了什么/发现了什么/达成了什么）
   - 失败描述要有后果（浪费了时间/发出了噪音/受伤了）

6. 如果是搜索 → 查掉落表投物资骰          → assets/items/物品与掉落表.md
7. 如果涉及NPC → 查好感度是否变化          → assets/npcs/
8. 跳到 → 第4步
```

---

### 第3C步：战斗处理

```
【进入战斗时】（仅第一轮执行）
1. 读取敌人数据                              → assets/enemies/丧尸图鉴.md
2. 判定是否伏击
   - 玩家先发现敌人？敌人先发现玩家？        → PER暗骰
   - 伏击方第一轮自动先攻且获得+15攻击
3. 投先攻：所有参战者 AGI+PER+1d10，排列行动顺序
4. 记录所有参战者战前状态
5. 描述战斗场景：敌人外观、数量、距离、地形、可利用物体
6. 创建战斗日志文件                          → logs/combat/

【每轮执行】
7. 按先攻顺序，处理每个单位的行动：

   玩家回合：
   a. 等待玩家声明行动
   b. 确认行动类型（主要/次要/自由，行动分类见附录A）
   c. 执行检定：
      攻击 → 1d100 vs 属性×10+技能×5
      命中 → 对方选择闪避/格挡/不防御
      闪避成功=全免 / 格挡成功=半伤 / 不防御=全伤
      伤害 = 武器骰 + STR修正(近战) - 护甲值（最低1）
   d. 使用异能 → 扣SP，应用效果（异能消耗见附录A）
   e. 使用道具 → 应用效果，消耗道具

   敌人回合：
   a. 根据敌人行为模式决定行动（图鉴中有描述）
   b. 执行攻击检定和伤害
   c. 玩家选择防御方式
   d. 【丧尸命中玩家时】→ 感染检定：1d100 vs CON×10
      失败=感染值+1（感染阶段效果见附录A）

8. 轮末结算（状态效果详见附录A）：
   □ 【倒计时】角色卡"活跃效果追踪"中所有效果剩余轮数-1
     - 归零的效果：移除并应用结束效果（如疲惫）
     - 新产生的效果（如疲惫）：写入追踪表
   □ 流血角色：-1d4 HP（急救止血）
   □ 中毒角色：-2 HP，所有检定-5（CON检定或解毒剂解除）
   □ 感染角色：按阶段应用属性减值
   □ 恐惧角色：所有检定-10（WIL DC15解除）
   □ 疲惫角色：物理检定-10，移动减半
   □ 燃烧：按情况处理
   □ 武器耐久-1（攻击方）/ 防具耐久-1（被命中方）
   □ EP消耗（格挡等）
   □ 更新角色卡的"活跃效果追踪"表

9. 输出本轮结束状态表（所有参战者HP和状态）
10. 如果战斗结束：
    - 计算经验值（经验值表见附录A）
    - 投战利品骰（如有）
    - 更新角色卡
    - 完成战斗日志
    - 跳到 → 第4步

如果战斗继续 → 回到第7步下一轮
```

---

### 第3D步：系统操作处理

```
根据玩家请求执行：

"查看状态"  → 读取角色卡，展示HP/SP/EP/感染/饥饿/位置/任务
"查看背包"  → 展示物品列表和负重
"查看日志"  → 读取对应日志文件的最近内容
"查看地图"  → 展示已探索区域列表和当前位置
"存档"      → 执行存档流程（见情境检查清单 §H）
"查看规则"  → 读取并解释对应规则

系统操作不消耗游戏内时间，不需要投骰。
处理完毕后等待玩家下一条行动指令，不执行第4-6步。
```

---

### 第4步：推进世界状态

行动处理完毕后，更新游戏世界：

```
1. 推进游戏内时间
   - 对话：1-5分钟
   - 搜索房间：10-30分钟
   - 战斗每轮：6秒
   - 移动到相邻区域：5-15分钟
   - 长距离移动：30-60分钟
   - 急救：5-15分钟

2. 检查时间触发器
   □ 累计超过30分钟 或 进入新区域？→ 投随机事件骰  → story/events/
   □ 距上次进食超过8小时？→ 饥饿等级+1（饥饿效果见附录A）
   □ 距上次饮水超过6小时？→ 口渴等级+1（效果比饥饿更严重，每级额外-5）
   □ 感染者到检定时间？→ 投感染恶化骰（间隔见附录A感染表）
   □ 手电筒还有电吗？→ 更新电量计时（满电4小时）

3. 检查事件触发
   □ 当前场景是否有预设触发条件被满足？      → scenes/ 对应文件
   □ 是否完成了任务目标？                    → story/main_plot/ 或 side_quests/
   □ NPC好感度是否跨过阈值？                 → assets/npcs/
```

---

### 第5步：更新文件

```
□ 角色卡：HP/SP/EP/物品/经验/状态有变化 → 写入 characters/active/
□ 系统日志：追加本轮所有检定和数值变动   → logs/system/
□ 会话日志：追加叙事内容（重要事件后）   → logs/session/
□ 战斗日志：如在战斗中，追加本轮数据     → logs/combat/
□ 探索日志：如进行了搜索/探索，记录发现  → logs/exploration/
□ 主线进度：如有推进，更新勾选            → story/main_plot/
```

### 第6步：组织输出

将以下内容组织成一条完整的回复发给玩家：

```
## 📍 [位置名]

[叙事描述]
  - 至少3-5句，覆盖至少3种感官（视觉+听觉+嗅觉/触觉/第六感）
  - 展示而非告知：不说"房间里有物资"，而是描写具体的物品细节
  - 包含暗示：至少1个明示 + 1个暗示（引导玩家自然地想去搜索/调查）
  - 如果有战斗/检定，将动作描写融入叙事中

### 🎲 检定结果（如有）
- 检定：[类型] | 投骰：1d100=[X] vs 目标值[Y] | 结果：[成功/失败]
（检定数据放在叙事之后，保持叙事的沉浸感不被打断）

### 📊 状态变化（如有）
- HP: X/X → Y/X (-Z)
- 获得物品：[物品名]
- 感染值：X → Y

### 📋 当前状态
- HP: XX/XX | SP: XX/XX | EP: XX/XX
- 位置：[当前位置]
- 游戏内时间：[HH:MM]

### ❓ 你可以...
1. [基于明示的行动——玩家一定会看到的选项]
2. [基于暗示的行动——仔细阅读描述后会注意到的选项]
3. [基于环境的行动——利用地形/通道/NPC的选项]
4. [自由输入其他行动]
```

**叙事描述的写作规范**（详见 `dm_guide/AI_DM指南.md` "叙事与文学规范"章节）：

1. **感官叠加**：不要只写"看到"，要写听到、闻到、感觉到
2. **暗示引导**：通过异常细节引导玩家注意可交互的对象
   - 明示："角落有一个翻倒的急救箱"
   - 暗示："所有柜门都开着，除了最右边那个"
3. **动作描写**：战斗和检定的结果要用动作叙述，数据放在叙事后面
4. **NPC个性**：NPC说话要有口癖、停顿、小动作，不能像机器人念台词
5. **日常错位**：用日常物品的异常状态制造不安感

> "叙事描述"和"你可以"两个区块每次都必须有。检定结果和状态变化按需出现。

---

### 第7步：验收清单 ✓

**在发出回复之前，逐条确认以下所有项目。任何一条未通过则回到对应步骤修正。**

```
━━ 行动合理性 ━━
□ 玩家的行动是否有他可能没意识到的严重后果？→ 是否已暂停提醒并获得确认？

━━ 规则正确性 ━━
□ 检定公式是否正确？（属性×10 + 技能×5 + 难度修正）
□ 伤害计算是否正确？（武器骰 + 修正 - 护甲，最低1）
□ 大成功(01-05)/大失败(96-00)是否正确处理？

━━ 遗漏检查 ━━
□ 丧尸命中了玩家？→ 是否做了感染检定？（1d100 vs CON×10）
□ 使用了武器/被命中？→ 是否扣了耐久？
□ 使用了异能？→ 是否扣了SP？SP为0是否投了副作用骰？
□ 异能有持续效果？→ 是否写入角色卡"活跃效果追踪"（含剩余轮数）？
□ 活跃效果追踪表中的剩余轮数是否-1了？归零效果是否移除并应用结束效果？
□ 行动产生了噪音？→ 是否投了噪音吸引骰？（隧道×2）
□ 角色有流血/中毒/感染等状态？→ 是否按附录A结算了？
□ 使用了瘟疫之触？→ 是否记录了感染值+0.5？

━━ 时间与生存 ━━
□ 游戏内时间是否推进了？
□ 是否到了饥饿/口渴检定点？（8h/6h）
□ 是否到了随机事件投骰点？（30分钟/新区域）
□ 感染者是否到了恶化检定时间？

━━ 文件更新 ━━
□ 角色卡是否更新了？（数值/物品/状态变化）
□ 系统日志是否记录了？（所有检定和变动）
□ 其他日志是否按需写入了？

━━ 输出完整性 ━━
□ 叙事描述是否覆盖了至少3种感官？
□ 叙事中是否包含至少1个明示暗示（引导玩家可交互对象）？
□ 战斗/检定结果是否有动作描写（不只是数据）？
□ NPC对话是否有个性化语气（不是信息播报）？
□ 回复是否提供了后续行动选项（含基于暗示的选项）？
□ 当前区域的特殊规则是否生效了？（隧道黑暗/西区污染等）
```

**全部通过 → 将本轮工作记录写入系统日志（格式见附录B）→ 发出回复。**

## 特殊场景补充流程

### 休息处理（玩家说"我要休息"时插入第3步和第4步之间）

```
1. 告知当前位置安全等级和预期恢复量
   安全=100%恢复,0%遇袭 | 较安全=75%,10% | 危险=50%,30% | 极危险=25%,60%
2. 玩家确认后：
   a. 计算恢复量
      - 1h：CON点HP + WIL点SP + (CON+STR)点EP
      - 4h+：50%最大HP/SP/EP
      - 进食饮水：+10HP+5EP
   b. 投遭遇骰（1d100 vs 上述百分比）
      - 触发 → 中断休息，按已完成时间比例恢复，进入战斗
   c. 更新饥饿/口渴计时
   d. 更新感染检定计时
```

### 场景切换（进入新区域时插入第4步）

```
1. 读取新场景文件                             → scenes/
2. 投随机事件骰                               → story/events/
3. 检查新区域特殊规则并告知玩家（见附录A区域规则表）
4. 描述新场景（视觉+听觉+嗅觉+温度）
```

## 附录A：内嵌规则速查（无需跳转）

> 以下是工作表执行过程中高频查阅的规则，直接嵌在这里避免反复读取其他文件。

### A1. 难度修正表

| 难度 | 修正 | 描述 |
|------|------|------|
| 轻松 | +30 | 几乎不可能失败 |
| 普通 | +10 | 稍有挑战 |
| 标准 | ±0 | 正常难度 |
| 困难 | -10 | 需要一定能力 |
| 极难 | -20 | 高手才能完成 |
| 噩梦 | -30 | 几乎不可能 |
| 奇迹 | -50 | 需要超凡运气 |

### A2. 技能-属性对照表

| 技能 | 关联属性 | | 技能 | 关联属性 |
|------|----------|-|------|----------|
| 近战格斗 | STR | | 机械 | INT |
| 射击 | AGI | | 电子 | INT |
| 格挡 | STR/CON | | 医学 | INT |
| 闪避 | AGI | | 博学 | INT |
| 急救 | INT | | 说服 | WIL/INT |
| 搜索 | PER | | 威吓 | STR/WIL |
| 潜行 | AGI | | 察言观色 | PER/WIL |
| 求生 | CON/PER | | 领导 | WIL |
| 锁技 | AGI/INT | | | |

### A3. 战斗行动分类

**主要行动（每轮1次）：** 攻击、使用道具、全力防御(闪避/格挡+20)、全力冲刺(移动×2弃攻击)、复杂异能
**次要行动（每轮1次）：** 换武器、快速检查、简短对话、简单异能、掩护移动
**自由行动：** 移动AGI×2米、倒地/站起、喊话

**借机攻击：** 从敌人近战范围(2m内)移动离开时，敌人可免费攻击1次（移动者不能防御）。全力冲刺/掩护移动/特定异能可避免触发。详见 `rules/01_核心规则书.md` §4.2.1

### A4. 状态效果完整表

| 状态 | 每轮/持续效果 | 解除条件 |
|------|--------------|----------|
| **流血** | -1d4 HP | 急救检定成功 |
| **中毒** | -2 HP，所有检定-5 | CON检定成功 或 使用解毒剂 |
| **恐惧** | 所有检定-10，不能靠近恐惧源 | WIL检定DC15成功 |
| **疲惫** | 所有物理检定-10，移动减半 | 休息1小时 |
| **眩晕** | 跳过下一轮行动 | 1轮后自动解除 |
| **倒地** | 被近战攻击+15，闪避-10 | 消耗次要行动站起 |
| **燃烧** | -1d6 HP | 消耗主要行动扑灭(AGI检定DC标准) |

### A5. 感染阶段完整表
